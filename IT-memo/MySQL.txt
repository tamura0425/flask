
グローバルレベルの権限を設定
grant all privileges on webdb1.* to hoge identified by 'hiro';
grant all on *.* to 'hiro'@'%';


SELECT user, host, plugin FROM mysql.user;


ユーザ名＆ホストの変更
ename user ユーザ名@ホスト名 to 新ユーザ名@新ホスト名

ename user hiro@% to hiro@localhost


mysqlで接続しているDBサーバのホスト名を確認する方法

show variables like 'hiro';

 ユーザーを削除

drop user ユーザー名@ホスト名;

drop user hiro;



ユーザーを作成する

CREATE USER '<ユーザー名>'@'localhost' IDENTIFIED BY '<パスワード>’;
GRANT ALL PRIVILEGES ON <db名>.* TO '<ユーザー名>'@'localhost';


権限の確認。

show grants for '<ユーザ名>'@'localhost';


取り消し
\c


MySQL8.0以降で接続できない場合は「認証プラグイン」を変更する

1. ユーザーの認証方式の変更

ユーザーの認証方式を旧式の「mysql_native_password」に変更する。
ALTER USER 'ユーザー名'@'localhost' IDENTIFIED WITH mysql_native_password BY 'パスワード';

ALTER USER 'hiro'@'localhost' IDENTIFIED WITH mysql_native_password BY 'hirotada';

変更の確認をする。

SELECT user, host, plugin FROM mysql.user;




MySQLの状態確認
sudo service mysql status


MySQLの起動
[CentOS7] systemctl restart mysql


MySQLの停止
[CentOS7] systemctl stop mysql
[Ubuntu] sudo service mysql stop


MySQLの再起動
[CentOS7] systemctl start mysql
[Ubuntu] sudo service mysql restart

 
MySQLログイン
mysql -u root 


MySQLの自動起動設定（enable）
[CentOS7] systemctl enable mysql
[Ubuntu] sudo chkconfig mysql on
